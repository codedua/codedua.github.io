Fundamental

  • WebGL (Web Graphics Library) is often thought of as a 3D javascript API. (An author argues that in reality WebGL is just a rasterization engine and it has features that help to implement 3D: https://webglfundamentals.org/webgl/lessons/webgl-2d-vs-3d-library.html. Theo quan điểm của anh ta, một thư viện 3D cần phải handle 3D cho mình, mình chỉ cần pass vào dữ liệu như tọa độ đỉnh là nó sẽ tự tính những tính toán còn lại như clip space… ví dụ như thư viện Three.js là một thư viện 3D. ta có thể define camera view và truyền các dữ liệu 3D vào là được. OpenGL cũng tương tự. Tuy nhiên, OpenGL ES và webGL thì không, ta phải sử dụng các kiến thức toán học như ma trận… để xử lý, xoay đối tượng 3D… nên OpenGL ES và webGL không nên được coi là một thư viện 3D mà chỉ coi là một engine để rasterize )

  • WebGL programs consist of control code written in JavaScript and shader code (GLSL) that is executed on a computer’s Graphics Processing Unit (GPU).

  • When we use the term “real-time graphics” we mean that rendering can be done in less than 1/30 of a second and the user can view smooth graphical animations. Ý tưởng là các operation tốn kém thì chỉ thực hiện 1 lần. vì vậy trong đồ họa máy tính sẽ có bước preprocessing được thực hiện 1 lần, sau đó là bước rendering. Vì vậy, code chương trình javascript sẽ được chia làm 2 phần, một là preprocessing hay setup, một phần là rendering. phần rendering sẽ được thực hiện nhiều lần, mỗi lần vẽ đối tượng là một lần thực hiện phần rendering. code js chạy trên CPU, nó ra lệnh có GPU render những gì và khi nào.

  • The context is a JavaScript object that stores all of the state and behavior of the graphics displayed in a canvas.
